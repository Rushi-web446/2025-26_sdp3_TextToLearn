1. Abstract

This report presents the design and implementation of an AI-Powered Personalized Course Generator, a web-based platform that automates the creation of structured educational content. The system leverages Large Language Models (LLMs) like Google Gemini and Groq (Llama-3) to transform a simple user prompt into a multi-module curriculum complete with lessons, descriptions, YouTube video recommendations, and interactive quizzes.

By utilizing a robust MERN (MongoDB, Express, React, Node.js) stack integrated with a Redis-backed BullMQ worker system, the platform ensures highly responsive user experiences even during computationally intensive AI generation tasks. This allows for asynchronous, parallel processing of course outlines and lessons, providing users with a seamless transition from a learning concept to a comprehensive study roadmap.

2. Introduction

2.1 Background

The rapid evolution of technology and the vast availability of information on the internet have made self-directed learning more popular than ever. However, the sheer volume of fragmented resources often leads to cognitive overload for learners. Creating a structured learning path manually—finding the right topics, ordering them logically, and souring high-quality materials—is time-consuming and difficult for beginners.

2.2 Problem Statement

Traditional online learning platforms often offer rigid, pre-defined courses that may not align with a learner's specific needs or background knowledge. Key issues include:

  • Fixed Curricula: Users cannot easily find or create courses for niche topics.
  • Resource Fragmentation: Learners spend significant time searching for relevant videos.
  • Lack of Structure: Self-taught learners often skip prerequisites.

2.3 Objectives

  • Develop a web platform that generates complete course outlines from a single sentence topic description.
  • Implement a deep-learning driven lesson generator that produces rich, pedagogical content.
  • Integrate YouTube API to automatically fetch relevant supplementary video content.
  • Utilize background workers (BullMQ) to handle long-running AI generation tasks.

2.4 Scope

The scope encompasses topic refinement, 4-6 module outline generation, deep textual lesson production (with mental models and MCQs), automated YouTube search integration, and a personalized learning dashboard across a modern React-based interface.

3. Literature Review

Personalized learning is a foundational goal in educational technology. Early systems relied on static rule-based engines that lacked natural language depth. The recent breakthrough in Large Language Models (LLMs) such as GPT-4, Llama, and Gemini has enabled dynamic curriculum synthesis by understanding semantic relationships between complex topics. Research in "Prompt Engineering" indicates that by enforcing strict JSON outputs and specific pedagogical personas (e.g., senior educator roles), AI can generate content that rivals human-made course outlines in speed and consistency. This project leverages these findings using Google Gemini 1.5 to solve the "structure gap" in modern ed-tech.

4. System Architecture

4.1 High-Level Architecture

Figure 4.1: High-Level MERN Stack with Redis/BullMQ Processing Pipeline

The system follows a modern decoupled architecture. The React frontend interacts with a Node.js API, which persists data in MongoDB. Heavy computational tasks (AI generation) are offloaded to Redis-backed queues managed by BullMQ, ensuring a non-blocking UI.

4.2 AI Course Generation Pipeline

1. **Extract**: LLM refines the user's prompt into a formal title/description. 2. **Outline**: The worker generates JSON based on the `outline.prompt` role. 3. **Validation**: Schema checks ensure the output matches our DB requirements. 4. **Worker Notification**: Lesson generation jobs are triggered in the background to fill in modules.

4.3 Data Flow

Data flows from the React home page to the Express API, then to the BullMQ producer. The BullMQ worker consumes the job, calls the Gemini/Groq API, and updates the MongoDB database. The frontend polls the backend for job status to provide real-time updates.

7.3 Frontend Design

The React application uses a component-based modular design:

  • Dashboard (Home.js): Feature-rich input center with real-time job status polling via `useJobProgress`.
  • Roadmap View: Tree-based visualizer for tracking progress across modules.
  • Learning Hub (Course.js): Lesson rendering engine with support for Markdown, Quiz interaction, and Video embedding.

8. Implementation

8.1 AI Engine - Curriculum Design

The `outline.prompt` acts as a "Senior Architect," requiring exactly 4-6 lessons per module and a logical progression from "fundamentals → application → mastery." It forces valid JSON output for direct database insertion.

8.2 AI Engine - Lesson Generation

The `lesson.prompt` uses a "Senior Educator" persona to ensure content reaches "top 1%" quality. Every lesson must include a "Deep Intuition" section (80-100 words) and "Common Mistakes" to help learners avoid typical pitfalls.

8.3 Background Processing

By using BullMQ, we separate the user's request from the long-running AI API calls. This allows the user to explore the course outline immediately while the lessons are "lazily" generated.

8.4 Backend Implementation

The Express.js backend handles route-level validation and acts as a producer for the Redis queue. It also provides a polling endpoint that allows the frontend to track the status of background jobs accurately using Mongoose logic.

8.5 Frontend - React SPA

The frontend is optimized for React 19, utilizing custom hooks like `useRecentCourses` and `useJobProgress` to maintain a dynamic and responsive UI state during AI generation.

9. Deployment

9.1 Manual Setup

1. Clone the repository and install dependencies. 2. Configure a local Redis instance and MongoDB cluster. 3. Set up Auth0 for authentication. 4. Run `npm run dev` to start the ecosystem.

9.2 Environment Configuration

The `.env` file must contain: `MONGO_URI`, `REDIS_CONNECTION`, `GEMINI_API_KEY`, `GROQ_API_KEY`, and `AUTH0_CLIENT_ID` for secure identity management.

10. Testing and Results

10.1 Performance Metrics

  • Outline Latency: ~8-12 seconds on average.
  • Lesson Generation: ~10-15 seconds per individual block.
  • Schema Reliability: 99.5% valid JSON extraction using Gemini 1.5 schema enforcement.

10.2 Optimization

Implementing high and low priority queues reduced lesson availability latency for the active user by over 70% compared to a single-threaded approach.
